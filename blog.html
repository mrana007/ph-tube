Discuss the scope of var, let, and const

var, let and const all are variable. var declarations are globally scoped. The var variable is an old method to declare a variable in javascript. var variable can be updated and we can re-declared var variable.
on the other hand let and const are blocked scope. In modern javascript, we use the let and const variable, which was introduced in the (ES6) update. let can be updated but we can not re-declared it. const variable can not updated also we can not re-declared const variable. now a days, the let and const variable is used more frequently in modern javascript as compared to the var variable.

Tell us the use cases of null and undefined

in null case variable has empty value. In JavaScript, null an object. When we assign null as a value to any variable, it means that it is empty or blank. It is to show that the variable has no value. When it gets assigned to a variable, it represents no value. Example: var A = null; here we assigned the value of A is null.
On the other hand in the case of undefined means variable is declared but the value is not assigned for the variable. In JavaScript, undefined is a type. When we do not pass an argument for a function parameter, the default value is taken as undefined. Besides, when a function doesn't return a value, it returns undefined. Example: var A;; here, we are not assigned the value of A. So the value undefined

What do you mean by REST API?

REST API stands for Representational State Transfer Application Programming Interface. There are different API styles, and each one has its unique architecture. One of the most common styles is REST. REST API is an architectural style characterized by client-server separation, HTTP interface, and statelessness
